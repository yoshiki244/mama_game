1. このゲームは何ができる？

ローグライク育成カードバトル「Project M(仮)」のプリプロ版です。

3つのエリア：草原（レベル上げ）→ 風雨の森（雨・中ボス「鬼」）→ 嵐の山頂（雷・ボス「ドラゴン」）。山頂のドラゴンは高レベルなので、草原で鍛えてから挑む構造です。

矢印の向き＝「使う側 → 使われる側」。

起動/画面切替/BGM/パーティ"] MAP[MapScreen] BATTLE[ProtoBattle] BUILD[BuildScreen] MENU[MenuScreen] PANEL["PanelModel ×人数"] STATS["PlayerStats ×人数"] SKILL["ProtoSkills"] ENEMY["ProtoEnemies"] PARTY["ProtoParty"] SAVE["ProtoSave"] AUDIO["ProtoAudio"] ART["ProtoPixelArt"] UI["ProtoUI"] MAIN --> MAP MAIN --> BATTLE MAIN --> BUILD MAIN --> MENU MAIN --> STATS MAIN --> PANEL MAIN --> SAVE MAIN --> AUDIO MAIN --> PARTY MAP --> ENEMY MAP --> ART MAP --> UI BATTLE --> SKILL BATTLE --> ENEMY BATTLE --> STATS BATTLE --> PANEL BATTLE --> ART BATTLE --> AUDIO BATTLE --> UI BUILD --> SKILL BUILD --> PANEL BUILD --> PARTY BUILD --> UI MENU --> STATS MENU --> SAVE MENU --> PARTY MENU --> ART MENU --> UI

設計の肝：

  • データ・道具は使われるだけの一方通行（GDDの「データ層→ロジック層→UI層」に沿う）
  • PanelModelとPlayerStatsは人数分存在（各員が自分の盤面・成長を持つ）
  • 敵・技・仲間を増やすときは図鑑に1行足すだけで画面側は無改造

4. 各ファイルの詳細

ProtoMain.cs — 劇場の支配人

ゲームに1人だけ。起動時にカメラ・Canvas・4画面・BGM・パーティを用意しセーブを復元。画面切替（ShowMap等）、パーティ管理（最大3人・各員に盤面と成長）、BGMの場面別自動切替（ボスは levelOffset で判定）。

MapScreen.cs — フィールドの管理人

72pxのマス目を1マスずつ移動。3エリアを LoadArea で組み替え（上端で次へ/下端で前へ）。敵は見えていて接触でバトル・徘徊あり。エリア2・3は雨、3は雷。仲間はプレイヤーの過去位置を辿って追従。

ProtoBattle.cs — バトルの審判＋演出家

最大ファイル。①開始＝メンバーごとに自分の盤面から山札を作り手札3枚 ②カード選択で点滅チャレンジ（正解で威力200%）③技ごとの専用モーション＋ヒット音＋数値ポップアップ ④ターン制/ATBの2方式 ⑤メンバー個別HP・戦闘不能 ⑥勝てば全員EXP。ボスは levelOffset ぶん格上。

BuildScreen.cs — パズル盤の店主

上部タブでメンバーを選び各自の盤面にピース配置（クリック/ドラッグ/Rキー回転）。盤面の形はメンバーで違う。占有マス数＝出現率%。データは PanelModel が持つ。

MenuScreen.cs — 受付窓口

矢印キー＋Enterで操作、選んだだけで右に詳細表示。ステータス（メンバータブ）/仲間（←→で増減）/設定（音量・BGM・バトル方式）/セーブ。

PanelModel.cs — 盤面の帳簿係

「どこに何のピースがあるか」だけを記録する純粋データ。形状マスク対応（キャラごとに有効マスが違い、どれも100マス）。配置可否・回転・山札生成を判定。

ProtoSkills / ProtoEnemies / ProtoParty — カタログ類

技＝名前・威力・形・色。敵＝HP・攻撃力・技5種・出現率・飛行フラグ・ボスのレベル補正（エリア別出現）。仲間＝MAMA/アカネ/ソラ＋各自の盤面形状（正方形/三角形/ひし形）。

PlayerStats.cs — 履歴書

レベル・EXP・HP・攻撃・防御・素早さ。EXPが貯まると自動レベルアップで全能力上昇。

ProtoSave.cs — 金庫番

全員のステータス・盤面・WaveをPlayerPrefsにJSON保存。起動時に自動ロード（旧形式互換あり）。

ProtoAudio.cs — お抱え作曲家

音楽ファイル不使用、波形を計算して生成。BGM4曲（フィールド/バトル/ボス/嵐）＋効果音（ヒット4段階・風切り・雷鳴）。

ProtoPixelArt.cs — お抱え絵師

文字マップを1文字＝1ドットで画像化。キャラ・敵6種・戦闘背景・見下ろしマップ（3テーマ）を生成。文字を書き換えるだけで編集可能。

ProtoUI.cs — 大道具係

ボタン・文字・ゲージを1行で作る共通関数集。黒縁取り・金見出しなどデザイン統一もここ。

7. Unityへの落とし込み方

シーン proto.unity には空のGameObject＋ProtoMainが1つあるだけ。▶を押すと Awake() が走り、カメラ・UI・キャラ・音をコードが全部その場で組み立てます。

メリットデメリット
シーンの手作業ゼロ、コードだけで全部変わるエディタで見た目を直接いじれない
「どこで何が作られるか」が全部コードにある本番ではアーティストが作業不可（プリプロ専用）
Unity用語使われ方
MonoBehaviour画面系ファイル。UnityがUpdate等を呼ぶ
Awake/Start/Update起動時1回／毎フレーム（移動・入力・雨・雷）
コルーチン(IEnumerator)「0.5秒待ってから次」。演出の順番制御
Sprite/Texture2D画像。ProtoPixelArtがドットで生成
AudioClip音。ProtoAudioが波形計算で生成
PlayerPrefs端末内の保存領域。セーブ置き場

動かす手順：①Assets/Scenes/proto.unityを開く ②▶再生（草原から開始）③WASD/矢印=移動、B=メニュー、バトルは数字入力＋Enter
