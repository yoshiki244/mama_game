Project M プロトタイプ ソースコード 詳細解説

各ファイルをメソッド（関数）単位まで掘り下げて解説します。

「概要を知りたい」なら ソースコード解説.md を、「中で実際どう動くか知りたい」ならこの資料を読んでください。

読む前に：C#とUnityの最低限の言葉

言葉意味（ざっくり）
クラス1つの「係」。ファイル＝クラスとほぼ同じ
メソッド（関数）係ができる「動作」。void 名前() { ... } の形
変数 / フィールド係が覚えておく「メモ」。_ 始まりは内部メモの慣習
public / private外から呼べる / そのファイル内だけ
void「結果を返さない動作」。bool なら成否を返す
IEnumerator ＋ yield時間のかかる処理。yield return 待つ で途中で一時停止できる（演出の要）
List<T>同じ種類のものを並べた「箱」。Addで追加
イベント Action「○○が起きたら呼んでね」の予約

ProtoMain.cs — 司令塔

ゲームに1個だけ存在。Awake()（Unityが起動時に1回呼ぶ）で全部を組み立てる。

主なフィールド（覚えていること）

  • Party（List）＝今のパーティ。Panels＝各員の盤面。MemberStats＝各員の成長。3つは同じ人数・同じ順番で並ぶ
  • Wave＝進行度（敵の強さに使う）。AtbMode／BgmEnabled＝設定

主なメソッド

メソッド何をする
Awake()カメラ→Canvas→4画面→パーティ復元→セーブ読込→BGM生成、の順で初期化
AddPartyMember() / RemovePartyMember()仲間を増減。Party・Panels・MemberStatsの3つを同時に増減させるのがポイント（ズレると事故る）
ShowMap/ShowMenu/ShowBuild()他画面をHide()して目的画面をShow()。画面はDestroyせず表示ON/OFFで切り替えるので高速
StartBattle(enemy)バトル開始。enemy.levelOffset>0ならボス曲、それ以外は通常バトル曲を鳴らす
OnBattleWon()Wave+1 → 倒した敵をマップから消す → マップへ戻る
PlayBgm(clip)同じ曲なら何もしない。起動初期化中(clip=null)は無視するガード入り（ここが抜けてて起動バグになった）
💡 設計の核：他のファイルは全部 _main（ProtoMain）を握っていて、_main.Party や _main.StartBattle() 経由で連携する。ProtoMainがハブ。

PanelModel.cs — 盤面の帳簿係（純粋データ）

10×10などの盤面で「どこに何のピースがあるか」だけを管理。見た目は一切持たないので、ビルド画面でもバトルの山札生成でも同じものを使い回せる。

中の構造

  • Placement（内部クラス）＝置かれたピース1個分。「どのスキルか＋占有マスのリスト」
  • _grid[x,y]＝各マスに置かれたPlacement（無ければnull）
  • _valid[x,y]＝そのマスが盤面として存在するか（キャラごとに形が違うため）

主なメソッド

メソッド何をする
PanelModel(bool[,] mask)形状マスク付きで生成。trueのマスだけが有効（三角形・ひし形を作れる）
IsValid(x,y)そのマスが盤面内かつ存在するか
Rotate(v, rot)（static）座標を90度回す数学処理。(x,y)→(y,-x) を rot 回
CanPlace(skill, anchor, rot)はみ出し・他ピースとの重なりをチェック
Place(...) / RemoveAt(x,y)配置 / 撤去
RotatePlacementAt(x,y)置いてあるピースをその場回転。盤外にはみ出たら内側へ自動スライドして救済
OccupiedCount() / ValidCount()埋まったマス数 / 有効マス総数（どの形も100）
BuildDeck()有効マスを上から順に並べて山札にする。ピースのあるマス＝そのスキル、空きマス＝null（通常攻撃）
💡 「1マス＝1枚のカード」がここで実現されている。盤面を組む＝デッキを組む、というGDDの核。

PlayerStats.cs — 履歴書

シンプル。レベル・EXP・HP・攻撃・防御・素早さを持つ。

メソッド何をする
GainExp(amount)EXPを足し、ExpToNext（=Level×50）を超えたらLevelUp()。1回で複数レベル上がることもあるので while ループ。レベルが上がったら true を返す
LevelUp()Level+1、HP+10・攻撃+2・防御+1・素早さ+1

ProtoSkills.cs / ProtoEnemies.cs / ProtoParty.cs — 図鑑類

データを並べるだけのファイル。ここを編集するのがゲームバランス調整の基本。

ProtoSkills

  • ProtoSkill＝名前・威力・色・shape（マスの並び＝ポリオミノ形状）
  • All＝6種のリスト。shapeの V(x,y) がマス1つ。形が複雑なほど点滅チャレンジが難しい

ProtoEnemies

  • EnemyAttack＝技1つ（名前・倍率・ヒット数・抽選の重み）
  • ProtoEnemy＝HP・攻撃力・サイズ・flying（飛行=影なし）・levelOffset（ボスの格上げ）・技5種
  • PickAttack(enemy)＝重み付き抽選。weightの合計から乱数を引いて技を選ぶ
  • RandomEnemy(area)＝エリアごとに出る敵を変える（草原は弱敵、山頂はゴーレムのみ等）

ProtoParty

  • Roster＝MAMA/アカネ/ソラ（髪色のみ違う）
  • BoardMask(memberIndex)＝各キャラの盤面形状を生成。widths配列（各行のマス数）から bool[,] を作る。例：アカネは{1,3,5…19}でピラミッド型＝合計100

ProtoSave.cs — 金庫番

SaveData（保存する内容の設計図）を JSON 文字列にして PlayerPrefs（端末内の保存領域）へ。

メソッド何をする
Save(main)全員のステータス＋全員の盤面配置＋Waveを集めてJSON化して保存
Load(main)JSONを読み戻す。membersがあれば新形式、無ければ旧形式（リーダーのみ）として復元＝昔のセーブも壊れない互換処理
⚠️ ピースは member（誰の盤面か）も一緒に保存。外したメンバーの盤面データは読込時に無視。

ProtoUI.cs — 大道具係（共通部品）

「ボタンを1個作る」を毎回20行書くのは大変なので、1行で作れる関数にまとめた道具箱。staticなので誰でも ProtoUI.CreateButton(...) と呼べる。

メソッド何を作る
CreateCanvas()UIの土台。画面サイズ追従の設定込み
CreatePanel(...)色付きの四角（背景・帯・マスなどに使う万能部品）
CreateText(...)文字。全テキストに黒縁取りを自動付与するのでどんな背景でも読める
CreateButton(...)押せるボタン（クリックで指定の動作）
CreateGauge(...) / SetGauge(...)HPバー等。SetGaugeで割合に応じて伸縮
CreateInputField(...)数字入力欄（点滅チャレンジ用）
StyleTitle(...)見出しを金色・太字・字間広めに（高級感の統一）

末尾の CellClickHandler＝マスの「左/右クリック・ドラッグ」を区別して受け取る部品（ビルド画面で使用）。

ProtoPixelArt.cs — お抱え絵師

画像ファイルを使わず、文字マップから画像を生成する。

仕組みの心臓 FromMap(rows, palette)

rows＝"..HHSS.." のような文字の行。1文字＝1ドット。palette＝文字→色の対応表。

これを1ピクセルずつTexture2Dに書き込んでSprite（画像）にする。未定義の文字はマゼンタで表示＝書き間違い発見器。

絵を作るメソッド

メソッド何の絵
Mama() / Mama(色...)MAMA。色違い版で仲間（アカネ/ソラ）も生成
MapMama(dir, frame, 色...)マップ歩行用ミニキャラ（4方向×2コマ）
Dragon() / Oni() / Bat() / Golem() / Slime()敵5種
NatureBackground()バトルの横スクロール背景（空・木・花）
TopDownField(...,theme,path)見下ろしマップタイル。themeで草原/森/山頂を切替、pathで石の階段を描く

ProtoAudio.cs — お抱え作曲家

これも素材ファイル不使用。音の波形を数式で計算してAudioClipを作る。

メソッド何の音
CreateBgm()フィールド曲（長調・のどか）
CreateBattleBgm()通常バトル（短調・2倍速・ドラム）
CreateBossBgm()ボス戦（重厚な行進曲）
CreateStormBgm()嵐の山頂（不穏なドローン）
CreateThunder() / CreateSwing() / CreateHitClip(tier)雷鳴／風切り音／ヒット音4段階
仕組み：Mathf.Sin(2π×周波数×時間)＝サイン波、Mathf.Sign(...)＝矩形波（ファミコン音）、Random＝ノイズ（打撃・雷）。melody配列の周波数を並べてメロディにしている。

MapScreen.cs — フィールドの管理人

状態を持つフィールド

  • _area（0草原/1森/2山頂）、_gridPos（プレイヤーのマス）、_facing（向き）
  • _enemies（マップ上の敵）、_trees（通行不可マス）、_followers（追従する仲間）

主なメソッド

メソッド何をする
LoadArea(area, spawnAt)エリアまるごと組み替え：敵を消す→地形(木/岩/階段)生成→環境(雨/雷/暗さ)設定→敵とボス配置→BGM切替
Update()毎フレーム：雨・雷を動かす→入力を読む→移動 or エリア移動 or エンカウント判定
StepTo(target)（IEnumerator）1マス歩くアニメ。同時に仲間を隊列で1マス前進させる（_trail＝過去位置の履歴を辿る）
UpdateEnemies()敵が数秒ごとに1マス徘徊。プレイヤーに踏み込むと向こうからエンカウント
UpdateLightning() / LightningStrike()エリア3で数秒おきに稲妻＋雷鳴＋画面フラッシュ
Encounter(enemy)（IEnumerator）「！」を出してStartBattleへ
Hide()画面を隠す。StopAllCoroutines()＋雷の残骸掃除（演出中に閉じてもゴミが残らない）
⚠️ コルーチンが途中で止まると破棄済みオブジェクトを触る → if (rt == null) yield break; のガードが各所に入っている（MissingReferenceException対策）。

BuildScreen.cs — パズル盤の店主

P（=_main.Panels[_member]）＝今編集中メンバーの盤面。タブで_memberを切り替える。

メソッド何をする
BuildBoard()メンバーの盤面の形に合わせてマスのマス目を生成。形の外（無効マス）は透明＆クリック不可
OnCellLeftClick(x,y)置いてあれば選択、空きなら選択中ピースを配置。置けなければエラー表示
OnCellBeginDrag/Drag/EndDragドラッグ移動。移動先を緑(置ける)/赤(置けない)でプレビュー、ダメなら元へ戻す
RotateAny()盤面で選択中のピース、なければトレイのピースを回転
RefreshBoard()盤面の色塗りと、下部の出現率内訳（通常攻撃○%＋各スキル）を更新

MenuScreen.cs — 受付窓口

メソッド何をする
Update()矢印で項目移動。選んだだけで右に詳細表示(_previews)。設定内は→で入り←→で値変更・Enterで次へ
ShowStatusTab() / RefreshStatusView()メンバータブで各員のステータス・立ち絵を表示
ShowPartyTab() / AddMember() / RemoveMember()仲間の確認と←→での増減
BuildSettingsContent()音量・BGM・バトル方式（ターン制/ATB）の行を作る
SaveGame()ProtoSave.Saveを呼ぶ

ProtoBattle.cs — バトルの審判＋演出家（最大ファイル）

状態

  • _enemy（敵）、_effWave（敵の実効レベル=Wave+levelOffset）
  • _hands（メンバーごとの手札）、_decks（メンバーごとの山札）、_acted（行動済みか）
  • _memberHP（メンバー個別HP）、_atb（ATBか）、_atbTimer（敵ゲージ）

流れの主なメソッド

メソッド何をする
Begin(enemy)バトル開始。HP計算・メンバー生成・山札と手札作成・敵の見た目差替・接地/影
Update()アイドルアニメ（呼吸）＋ATBの敵ゲージを進める＋Escで逃走
OnCardClicked(member, index) → PlayCard(...)カード使用。スキルなら点滅チャレンジ→倍率→ダメージ→演出
RunChallenge(skill)（IEnumerator）ピースのマスを光らせ→数字入力を待ち→ズレで倍率決定（0=200%)
AttackMotionFor(skill)技ごとの専用モーション（火球/居合/急降下…）に分岐
Impact(...)ヒット時：エフェクト＋ヒット音＋ダメージ数値＋シェイク。倍率で派手さが変わる
EnemyAttackSequence()敵の攻撃。技抽選→連続ヒット→狙われた本人の防御/回避で計算→個別HP減少
AtbEnemyAct()ATBでゲージ満タン時の敵攻撃
💡 演出が連続して順番に流れるのは、ほぼ全部 IEnumerator＋yield return new WaitForSeconds(...) の力。「光らせる→0.5秒待つ→消す→入力待ち…」を1本の流れで書けている。

まとめ：機能を足したいときの動き方

足したいもの触る場所画面側の改修
新しい敵ProtoEnemies.All に1行＋ProtoPixelArtに絵不要
新しい技ProtoSkills.All に1行不要（モーションを凝るならProtoBattleに追加分岐）
新しい仲間ProtoParty.Roster＋BoardMask不要
新しいエリアMapScreenのLoadArea/配色/出現テーブルLoadArea内で完結
新しいBGMProtoAudioにCreate関数＋ProtoMainで呼ぶ不要

データとロジックが分離しているので、多くの拡張は「図鑑に1行」で済む。 これがこのプロトの一番おいしい設計です。
